# ripgrep is faster than {grep, ag, git grep, ucg, pt, sift}

## TL;DR
本文介绍 Rust 编写的命令行搜索工具 `ripgrep`，旨在替代现有工具。它结合易用性与高性能，解析了其利用有限自动机、SIMD 算法及增量搜索实现极速的技术原理，并论证了其在速度与正确性上的显著优势。

## Summary
*   **核心主题**
    介绍命令行搜索工具 `ripgrep` (`rg`)，阐述其结合 `The Silver Searcher` 的易用性与 `GNU grep` 的高性能的设计理念，并通过技术解析与基准对比论证其在搜索速度、正确性及 Unicode 支持上的优势。

*   **关键信息**
    *   **工具定位与特性**：
        *   `ripgrep` 是一款跨平台（Linux, Mac, Windows）、用 Rust 编写的搜索工具，旨在替代 `grep`, `ag`, `git grep` 等现有工具。
        *   默认行为智能：自动递归搜索，默认遵守 `.gitignore` 规则，忽略隐藏文件和二进制文件。
        *   功能丰富：支持按文件类型过滤（如 `-tpy`）、多模式搜索、上下文显示、颜色高亮及 PCRE2 正则引擎（通过 `-P` 启用）。
        *   编码与格式支持：默认开启 Unicode 支持且不牺牲性能；支持 UTF-16, GBK 等多种编码；支持直接搜索压缩文件（gzip, xz 等）。
    *   **核心性能主张**：
        *   在单文件和大目录搜索中，性能与正确性均优于对比工具。
        *   是唯一在不导致严重性能损失的情况下提供完整 Unicode 支持的工具。
        *   反驳了“搜索多文件时使用内存映射更快”的观点，证明增量搜索通常更高效。
    *   **局限性（Anti-pitch）**：
        *   非 POSIX 标准工具，非所有系统预装（不如 `grep` 普适）。
        *   可能存在未覆盖的小众功能或特定边缘性能问题。
    *   **技术实现原理**：
        *   **正则引擎**：基于 Rust 的 regex 库，使用有限自动机，避免 PCRE 风格的回追踪，防止指数级时间消耗。
        *   **字面量优化**：自动提取正则中的前缀/后缀字面量，利用高度优化的 `memchr`（通常编译为 SIMD 指令）和 Boyer-Moore 算法快速定位候选匹配；对多模式使用 Aho-Corasick 或 SIMD 加速的 Teddy 算法。
        *   **搜索机制**：摒弃逐行搜索，采用增量缓冲搜索，在减少系统调用和内存开销的同时，高效处理上下文显示和行计数。
        *   **并发与 I/O**：使用最小化系统调用的递归目录迭代器；利用无锁 Chase-Lev 工作窃取队列分配任务；搜索结果先存入内存缓冲区，最后由单线程序列化输出，以维持最大并行度。

*   **逻辑脉络**
    1.  **引言与概述**：介绍 `ripgrep` 的背景，提出其在速度和功能上超越竞品的核心论点，并列出 25 个基准测试的验证计划。
    2.  **功能介绍**：通过“Pitch”和“Anti-pitch”详细阐述工具的优势、适用场景及局限性，并提供安装方法与基础用法示例。
    3.  **技术深度剖析**：从“背景、文件收集、搜索机制、打印输出”四个维度，深入解析搜索工具的内部架构，重点说明 `ripgrep` 如何通过算法选择（有限自动机、SIMD）、并发模型（无锁队列）和 I/O 策略（增量搜索）实现高性能。
    4.  **基准测试铺垫**：简要介绍基准测试的方法论、运行环境及测试用例分类（代码搜索、单文件搜索等），为后续具体数据展示做准备。
