## Summary
核心主题：使用 GitHub-Copilot 进行 Vibe-Coding 的经验总结与最佳实践

关键信息与逻辑脉络：

1.  项目背景
   - 作者使用 GitHub-Copilot 完成了两个小项目：push.bboy.app 和 cal.bboy.app
   - 目的是探索 AI 辅助编程的实际应用和效果

2.  开发环境与流程
   - 环境隔离：使用虚拟机或容器进行代码编写，避免污染宿主机
   - 开发发布流程：明确需求 → 编写代码 → 创建版本 Tag → 流水线构建 Docker 镜像 → 手动触发部署
   - 远程操作：使用 Termius SSH 配合 tmux 保持会话

3.  核心开发理念
   - 产品思维：做减法比做加法更重要，专注核心功能而非堆砌功能
   - 优先级：优化细节和保证代码安全性 > 追求功能数量
   - 工作方式：
     * 先有产品文档，再写代码
     * 每次只做一个任务，保持范围可控
     * 完整测试至关重要，测试重要性可能超过开发本身

4.  Copilot 使用技巧
   - 节省 Token 的方法：
     * 选择合适的模型（简单任务用轻量模型）
     * 使用 `/compact` 压缩上下文
     * 使用 `/clear` 清理不再需要的上下文
     * 用 `@` 引用具体文件，避免整个项目塞入上下文
     * 创建 `.github/copilot-instructions.md` 预定义规范
     * 先让 AI 列出执行计划，再动手实现

5.  开发者角色转变
   - 仍然需要懂代码，但角色转变为架构师
   - 负责技术栈选择、系统架构设计、接口定义和代码审查
   - 细节实现可交给 AI 完成

6.  风险防范
   - 代码备份至关重要
   - 破坏性 git 操作前必须先做备份
   - 分享了因误用 `git filter-repo` 导致代码丢失的血泪教训

7. 总结与建议
   - GitHub-Copilot 能大幅提升开发效率
   - 产品思维、安全意识和清晰需求是项目成功核心
   - 专注主要功能，做减法，优化细节和性能，注重安全
