## TL;DR
本文旨在通过制定涵盖错误处理、记忆管理及工作流优化的系统化规则，提升 OpenClaw AI Agent 的稳定性与效率，解决幻觉及操作失误问题，并提供实战避坑指南。

## Summary
核心主题
提升 OpenClaw AI Agent 的稳定性、智能化水平及使用效率，通过制定系统化规则解决“幻觉”及操作失误问题。

逻辑脉络
从实际使用痛点出发，确立通过“规矩”约束 Agent 的核心思路，进而详细制定涵盖错误处理、记忆管理、工作流优化及技术配置的具体规则，最后通过实战经验总结避坑指南，强调持续迭代的重要性。

关键信息

1. 核心设定原则
   *   **身份定义**：明确 Agent 的身份、风格、核心原则及能力范围，并根据需求动态调整。
   *   **规则固化**：确保新规则被 Agent 记忆，通过重复使用提升熟练度。
   *   **数据备份**：规定每次更新记忆库后必须执行 Git 提交并推送到远程仓库。

2. 工作规则与最佳实践
   *   **错误处理**：优先自动修复或切换方案，修复失败后再向用户报告，禁止遇错即停等待指令。
   *   **时间片管理**：长任务分段执行，每块控制在 30 秒以内，防止超时并展示进度。
   *   **Git 规范**：工作区更新必须自动提交至 `origin main`，配置固定的用户名与邮箱。
   *   **工具配置**：安装 web_search、web_fetch、message、sessions_send、exec 等必备工具。
   *   **记忆管理**：
        *   `MEMORY.md`：存放长期规则、配置及重要决策。
        *   `memory/YYYY-MM-DD.md`：存放每日操作日志与临时信息，定期整理。

3. 流程与交互优化
   *   **工作流优化**：采用批量处理原则，一条消息完成多个简单任务；针对模糊输入主动列出选项供用户选择。
   *   **响应策略**：关键信息（错误、警告）立即输出，长内容分批返回，先给摘要。
   *   **上下文预加载**：根据关键词（如“博客”、“部署”）提前读取相关配置，减少等待时间。
   *   **智能缓存与复用**：缓存搜索结果（30分钟有效）及常用文件；复用已生成代码或相似请求结果，减少重复工作。

4. 目录结构与自动化
   *   **目录规范**：所有脚本文件统一存放在 `scripts` 目录，保持根目录整洁。
   *   **自动化规则**：
        *   用户发消息前先读取 `MEMORY.md` 获取上下文。
        *   遇到有解的未知问题，将答案写入 `MEMORY.md`。
        *   重要决策记录于当日日志文件。
        *   网关重启后必须向用户报告状态。

5. 避坑指南
   *   **通信工具选择**：避免使用 QQ Bot，因其回复规则包含大量冗余格式化文本，导致 Token 消耗过大；推荐使用钉钉或飞书。
   *   **持续迭代**：规则与流程需根据实际使用场景不断更新，追求最适合自身需求的配置。
