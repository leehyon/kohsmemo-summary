# AI should help us produce better code
- URL: https://simonwillison.net/guides/agentic-engineering-patterns/better-code/
- Added: 2026-03-12 03:06:50
- Tags: #agent

## TL;DR
AI 可帮助开发者提升代码质量，避免技术债务。通过异步工具处理重构任务，扩展技术选择，并实现持续优化，使同时开发新功能和改进质量成为可能。

## Summary
核心主题：AI 应帮助开发者编写更好的代码，而非降低代码质量

关键信息与逻辑脉络：

1.  核心观点
   *   担忧：开发者担心使用 AI 工具会导致代码质量下降
   *   反驳：降低代码质量是选择问题，而非必然结果
   *   主张：应利用 AI 主动提升代码质量，避免技术债务

2.  避免技术债务
   *   技术债务定义：为节省时间而采取的"捷径"，未来需偿还
   *   最佳策略：从根本上避免产生技术债务
   *   常见技术债务类型：
     *   API 设计缺陷导致多处修改
     *   早期命名不当导致后续清理困难
     *   系统中产生重复但略有不同的功能
     *   单个文件过大需要拆分

3.  AI 在消除技术债务中的应用
   *   理想场景：AI 擅长处理概念简单但耗时长的重构任务
   *   实施方式：使用异步 AI 编码工具（如 Gemini Jules、OpenAI Codex、Claude Code）
   *   工作流程：
     *   启动 AI 代理，指定任务
     *   在分支或工作区中执行
     *   通过 Pull Request 评估结果
     *   根据质量决定采纳、修改或丢弃
   *   成本效益：代码改进成本极低，可容忍微小代码异味

4.  AI 扩展技术选择范围
   *   帮助发现解决方案：LLM 可提示开发者未注意到的常见解决方案
   *   探索性原型制作：
     *   技术选择验证：通过原型证明技术适用性
     *   AI 的作用：根据提示快速构建模拟系统进行负载测试
     *   成本优势：实验成本几乎为零，可并行测试多种方案

5.  复合工程循环
   *   概念来源：Every 公司的"复合工程"实践
   *   核心机制：基于每次执行结果迭代优化指令
   *   关键步骤：项目完成后进行回顾（复合步骤），记录有效方法
   *   最终目标：持续提升代码库质量，小改进产生复利效应
   *   双赢局面：AI 使同时发布新功能和提升质量成为可能
