# 胶水编程：业务需求出码最佳实践
- URL: https://www.bestblogs.dev/en/article/1b54037f
- Added: 2026-03-31 09:50:19
- Tags: #agent #best-practice #vibe-coding

## TL;DR
天猫提出“胶水编程”，构建四层物料体系，将 AI 转变为装配工。通过“能抄不写”策略，代码采纳率升至 97.9%，实现了高匹配度场景下的高效与高质量交付。

## Summary
**核心主题**
天猫团队提出的“胶水编程”实践，即利用 AI 高效组装现有业务模块以响应需求，而非从零创作。该方法通过构建四层物料体系，将代码采纳率从 50% 提升至 97.9%，验证了在特定高匹配度场景下，让 AI “抄代码”比“写代码”更能保证代码的可用性、一致性及交付效率。

**关键信息与逻辑脉络**

*   **核心理念与范式转变**
    *   **定位转变**：将 AI 从代码的“作者”转变为“装配工”。核心逻辑是“能抄不写，能连不造”，利用 LLM 擅长拟合而非无中生有的特性。
    *   **管控逻辑**：SPEC 管意图（做什么），物料管执行（怎么做、用什么做）。只有在意图明确且物料充足的情况下，AI 才能产出可控的高质量代码。
    *   **分层演进**：从 Vibe Coding（仅能用）到 SPEC Coding（写得对），最终进阶到 Glue Coding（写得像），实现企业级可控编码。

*   **四层物料体系架构**
    通过分层注入策略，为 Agent 提供精准的上下文：
    1.  **开发规范（规矩层）**：
        *   **内容**：定义不可逾越的约束，如依赖包路径、请求架构、组件选型等。
        *   **加载方式**：静态注入。通过云端配置自动生成 AGENTS.md，打开仓库时自动加载，确保规则始终在场。
    2.  **代码模式（骨架层）**：
        *   **内容**：提供可运行的代码样板间（如标准列表页结构），而非抽象规则。
        *   **加载方式**：静态注入。Agent 参照样板间复制结构，仅替换业务字段，解决代码风格和组织结构差异问题。
    3.  **领域知识（知识层）**：
        *   **内容**：覆盖内部私有组件文档、平台约束及踩坑经验（60%-70% 为物料文档）。
        *   **加载方式**：动态检索。通过 MCP 协议按需调用，避免上下文过载，填补模型对内部私有知识的盲区。
    4.  **任务规格（方案层）**：
        *   **内容**：具体需求的技术方案。针对不同复杂度（如状态流转、表单联动）提供 6 种 SPEC 模板，明确“这次具体做什么”。
        *   **加载方式**：动态生成。开发者填写模板生成 spec.md，定义业务差异点，指导 Agent 在骨架上编写“胶水代码”。

*   **技术实现与落地流程**
    *   **分层注入机制**：静态加载保证规则和模式不缺失；动态检索保证知识精准匹配。
    *   **工程载体**：以 VS Code 插件为主，集成 Agent Hooks 自动识别业务域并下发配置，实现打开仓库即生效。
    *   **全链路协作**：需求进入后，系统自动加载规范与模式，Agent 按需检索知识，结合 SPEC 生成代码，最终产出基于样板间骨架且符合规范的代码。

*   **维护与迭代机制**
    *   **物料飞轮**：每次交付后，踩坑经验自动沉淀为领域知识，优秀实现提炼为代码模式，形成复利效应。
    *   **上下文延续（Track 机制）**：每个需求的 spec.md 持久化存储在 .ai/tracks/ 目录下。后续 AI 编码时可读取历史上下文，理解前人决策，避免重复造轮子或风格割裂。
    *   **规格保鲜**：提供轻量级同步 Skill，通过手动触发、AI 辅助的方式修正代码与 spec.md 的差异，平衡自动化与人工确认成本。

*   **实践效果与误区规避**
    *   **成效数据**：采纳率从 ~50%（仅知识库）提升至 ~76%（加规范），最终达到 90%+（四层齐全）。后端工程师亦可利用此体系快速交付前端需求。
    *   **常见误区**：
        *   物料越多越好：实际上精准度比数量更重要，低效知识会干扰 Agent。
        *   样板间一次成型：需随架构升级同步维护，否则会产生过时代码。
        *   重物料轻 SPEC：SPEC 定义业务意图，物料定义技术实现，两者缺一不可。

**总结**
胶水编程的核心不在于模型能力的强弱，而在于团队知识资产的显性化与结构化。通过“90% 抄 + 10% 写”的策略，将 AI 的能力聚焦于代码组装与差异点实现，实现了业务需求的高效、高质量交付。
