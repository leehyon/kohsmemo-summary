# 为什么你的 AI Agent 总是像个智障
- URL: https://tonybai.com/2026/03/18/why-ai-agents-act-stupid-manus-expert-pitfall-guide/
- Added: 2026-03-19 02:07:27
- Tags: #best-practice #agent

## TL;DR
AI Agent 表现不佳源于工具选择认知过载，建议采用简化的 Unix 命令行交互，利用大模型对 CLI 的熟悉度，通过渐进式发现和错误引导实现高效任务处理。

## Summary
核心主题：本文探讨了 AI Agent 表现不佳的原因，并提出了一种基于 Unix 命令行的简化解决方案。

关键信息：
1.  问题根源：当前主流的 Agent 框架通过提供大量复杂的 Function Calling 工具，导致大模型面临"认知过载"，在众多工具中难以正确选择，从而表现"智障"。
2.  核心观点：原 Manus 后端技术负责人 MorroHsu 提出，应摒弃繁琐的类型化函数调用，转而使用简单的 Linux 命令行作为大模型与系统的交互接口。
3.  理论基础：大模型的训练数据中包含了大量 CLI 命令，使其天生熟悉命令行操作。Unix 哲学的"一切皆文件"和"管道"设计，完美契合大模型文本输入输出的本质。
4.  实践技巧：
    *   渐进式发现：不一次性提供所有命令信息，引导大模型按需探索。
    *   错误引导：将命令错误转化为带有指导意义的提示信息。
    *   双层架构：在执行层和大模型层之间建立"二进制守卫"和"截断溢出守卫"，防止上下文被污染。
5.  总结：化繁为简是架构的最高境界，将 Unix 哲学与大模型能力结合，比复杂的 API 接口更有效。

逻辑脉络：
1.  提出问题：AI Agent 在面对复杂任务时表现不佳，常被归咎于 Prompt 或模型能力不足。
2.  引入观点：引用 Manus 大佬的经验，指出问题根源在于工具选择的认知过载。
3.  核心解决方案：用单一的 run(command) 函数替代复杂的工具集，利用大模型对 CLI 的熟悉度。
4.  理论支撑：解释 Unix 哲学如何完美解决 Agent 的编排难题，特别是管道组合能力。
5.  实施细节：提供三个"防智障"导航系统设计技巧，确保简化方案的可行性。
6.  总结升华：强调最前沿的 AI 需要最古老的系统智慧，简化才是高级架构的体现。
