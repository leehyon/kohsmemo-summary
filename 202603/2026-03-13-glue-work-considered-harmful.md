# Glue work considered harmful
- URL: https://www.seangoedecke.com/glue-work-considered-harmful/
- Added: 2026-03-13 02:24:28
- Tags: #career

## TL;DR
"Glue work" 对团队效率至关重要，但公司晋升机制常忽视。工程师应战略性地而非持续性地进行此类工作，专注于自己负责的项目，而非提高整体效率。

## Summary
核心主题: "Glue work"（粘合工作）虽对团队效率至关重要，但在公司晋升和奖励机制中常被忽视，工程师应战略性地而非持续性地进行此类工作。

关键信息与逻辑脉络:

1.  "Glue work" 的定义与价值
   - 由 Tanya Reilly 在 2019 年提出
   - 指团队为提高效率所需的非 glamorous 工作，如更新文档、处理技术债务、入职培训、跨团队沟通等
   - 实用型工程师倾向于此类工作，因其明显有用，但在晋升时被忽视

2.  "Glue work" 与项目交付的关系
   - "Glue work" 是成功交付项目的关键，纯技术型黑客（pure hackers）无法有效交付
   - 需要处理大型组织中的摩擦才能创造价值

3.  公司不奖励 "Glue work" 的原因
   - 公司不希望你优先处理此类工作，而是希望你交付功能
   - "Glue work" 很难，公司希望你利用这项能力交付项目而非提高整体效率

4.  核心问题与公司期望
   - 工程师不应自行决定公司需要什么，而应执行公司领导层的使命
   - 以 60% 的效率执行公司使命优于花时间提高整体效率
   - 让团队适应公司的基本效率水平比短暂地消除摩擦更可持续

5.  "Glue work" 的战略性应用
   - 应为你负责的项目进行 "Glue work"，确保项目成功
   - 不会因 "Glue work" 本身获得奖励，但会因项目成功获得奖励
   - 对其他项目，只需完成常规工作

6.  大型科技公司的效率与增长逻辑
   - 大型科技公司效率约为 20-60%，效率随规模扩大而降低
   - 公司增长是为了捕获更多"表面面积"，即使效率较低也能创造更多价值
   - 员工通过 "Glue work" 将团队效率提升到 80-90% 是公司接受的"免费价值"，但无意长期锁定，因其不可持续

7.  对特殊人才的建议
   - 具备 "Glue work" 能力的工程师可以战略性地利用这一能力成为更有效的工程师
   - 不应一直这样做，除非你真的愿意并清楚后果
