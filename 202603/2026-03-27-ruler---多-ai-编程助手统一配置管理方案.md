# Ruler - 多 AI 编程助手统一配置管理方案
- URL: https://atbug.com/ruler-unified-ai-coding-assistant-configuration/
- Added: 2026-03-27 05:29:28

## TL;DR
Ruler 通过单一数据源集中管理多 AI 编程助手的配置，实现一键分发，解决配置碎片化问题。

## Summary
**核心主题**
Ruler 是一个旨在解决多 AI 编程助手配置碎片化问题的 CLI 工具，通过建立“单一数据源”实现 Instructions、MCP 服务器配置及 Agent Skills 的集中管理，并一键分发至 30+ 种 AI 工具。

**关键信息**

1.  **背景与痛点**
    *   AI 编程工具（如 Kilocode、Claude、Copilot）爆发，开发者需频繁在多个工具间切换。
    *   各工具的 Instructions 文件位置、MCP 配置格式、Skills 目录结构及特定配置项均不统一。
    *   现有的 AGENTS.md 标准仅统一了指令文件名和位置，未能解决 MCP 配置分散、Skills 管理混乱及缺乏同步机制的问题。

2.  **核心理念与机制**
    *   **单一数据源**：所有配置集中存储在项目根目录的 `.ruler/` 文件夹内，包含主指令文件（如 AGENTS.md）、配置文件（ruler.toml）及 Skills 目录。
    *   **自动分发**：通过执行 `ruler apply` 命令，自动将配置生成并写入到各个工具对应的专属文件中（如 `.cursor/mcp.json`、`.claude/skills/`）。
    *   **自动维护 .gitignore**：自动将生成的配置文件添加到 .gitignore，保持项目仓库整洁，仅提交 `.ruler/` 源文件。

3.  **关键特性**
    *   **集中式规则管理**：自动发现并合并 `.ruler/` 目录下的多个 Markdown 文件，支持自定义加载优先级。
    *   **嵌套规则加载**：支持 Monorepo 或复杂项目的分层配置，允许子目录拥有独立的 `.ruler/` 规则并与全局规则合并。
    *   **MCP 配置统一**：在 `ruler.toml` 中统一定义 MCP 服务器，自动适配并分发到不同工具的配置格式，支持 merge 或 overwrite 策略。
    *   **Agent Skills 分发**：集中管理 Skills 文件夹，自动将其复制到各工具所需的特定目录中。

4.  **安装与使用**
    *   **安装**：支持通过 npm 全局安装或使用 npx 直接运行，要求 Node.js 版本 ^20.19.0 或更高。
    *   **初始化**：运行 `ruler init` 创建默认配置目录和文件。
    *   **应用配置**：运行 `ruler apply` 应用所有配置，或通过参数（如 `--agents`、`--mcp`、`--nested`）指定分发范围。

5.  **对比与适用场景**
    *   **vs AGENTS.md**：AGENTS.md 适合仅使用 1-2 个工具且无复杂配置需求的简单项目；Ruler 适合需频繁切换工具、使用 MCP/Skills、管理 Monorepo 或团队协作的复杂场景。
    *   **最佳实践**：结合使用 AGENTS.md 标准和 Ruler 管理，将 `.ruler/` 目录纳入版本控制，利用 NPM scripts（如 postinstall）实现团队成员环境的自动化配置。
