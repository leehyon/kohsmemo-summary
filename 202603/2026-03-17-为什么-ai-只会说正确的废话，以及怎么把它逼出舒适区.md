# 为什么 AI 只会说正确的废话，以及怎么把它逼出舒适区
- URL: https://yage.ai/context-infrastructure.html
- Added: 2026-03-17 06:25:46
- Tags: #agent #best-practice

## TL;DR
AI 产出平庸因训练机制倾向共识。突破需构建个人上下文系统，积累数据分层提炼，形成非共识判断，实现深度思考。

## Summary
核心主题：AI 产出平庸的原因及突破方法

关键信息与逻辑脉络：

1.  AI 产出平庸的现象与原因
   -  现象：AI 通常输出"正确的废话"，缺乏深度洞察
   -  原因：LLM 训练机制决定其默认输出为共识（consensus）
     -  next token prediction 机制倾向于选择概率最高（最多人认同）的输出
     -  RLHF 安全对齐进一步惩罚有争议、有立场的输出，鼓励平衡全面回答
     -  结果：LLM 回归均值，难以产生非共识的深刻判断

2.  突破共识天花板的认知基础
   -  深度 vs 广度：真正的深度来自认知不对称，而非信息不对称
   -  AI 的局限性：可将小白提升到平均水平，但对平均水平以上的人增量有限
   -  转型点：AI 已从 CPU Bound（模型智能）走向 Memory Bound（上下文密度）

3.  个人上下文系统的构建方法
   -  大量积累：采集客观行为数据而非主观描述
     -  包括录音转写、会议记录、对话历史等真实决策场景数据
     -  AI 作为旁观者帮助提取模式，建立 context density 基础
   -  分层提炼：从原始数据中提取稳定信号
     -  L1 Observer：扫描文件变动，提取日常观察
     -  L2 Reflector：合并重复、清理过期、识别跨项目模式
     -  L3 Axiom：从稳定模式中蒸馏决策原则，而非事实
   -  按需加载：根据任务类型选择性加载相关上下文
     -  类似 CPU 内存层级结构（L1 cache、L2 索引、L3 具体文件）
     -  每个 skill 包含特定任务相关的 axiom 和判断标准

4.  系统运行与价值循环
   -  知识产品涌现：系统在消费 context 同时产生新 context
   -  自我认知进步：将隐性 bias 显性化，了解自己的优先级和倾向
   -  偏见的双重性：有品味的 bias 是深度来源，但也可能是质量不高的偏见

5.  实施建议与开源参考
   -  无捷径可走：无法通过简单修改 prompt 或更换模型突破
   -  开源参考实现：提供完整结构参考，但需个性化构建
   -  核心理念：注入个人非共识视角，将硅基大脑与人类灵魂结合
