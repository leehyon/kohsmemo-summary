## TL;DR
通过硬路由桥接解决双 NAT，结合 MosDNS + AdGuard Home 实现 DNS 分流，优化 OpenClash 配置，实现全屋代理与精确分流。

## Summary
核心主题：家庭网络环境改造，解决双 NAT 问题、设备监管不足、DNS 解析成本高及硬路由故障问题，实现全屋代理与精确分流。

关键信息：

1. 问题根源：
   - 硬路由作为二级路由导致双 NAT 转换
   - 软路由无法监管所有网络设备
   - OpenClash Fake-IP 模式 DNS 解析成本高
   - Linksys 路由器断电后红灯问题

2. 网络桥接解决方案：
   - 将硬路由改为桥接模式解决双 NAT 问题
   - 通过 DHCP 静态分配替代 MAC 绑定实现设备管控
   - 配置步骤：取消动态 DHCP，添加静态地址分配

3. 在线用户监控：
   - 安装 onliner 插件实现设备实时监控
   - 需下载 all.ipk 版本避免依赖问题

4. DNS 分流优化：
   - 使用 MosDNS 实现国内/外域名分流解析
   - 国内域名走本地 DNS，国外域名走代理
   - 配置 MosDNS 监听 5335 端口，通过 AdGuard Home 劫持流量

5. OpenClash 调整：
   - 取消绕过中国大陆 IP 选项
   - 停用本地 DNS 劫持
   - 配置自定义上游 DNS 服务器
   - 添加防火墙规则优化流量处理

6. 完整流程：
   - 硬路由桥接解决双 NAT
   - MosDNS + AdGuard Home 实现 DNS 分流
   - OpenClash 配置代理规则
   - 防火规则优化确保精确流量控制

逻辑脉络：
1. 问题识别 → 2. 网络架构调整（桥接）→ 3. 设备管控实现（DHCP 静态分配）→ 4. DNS 分流方案（MosDNS + AdGuard Home）→ 5. 代理工具优化（OpenClash）→ 6. 防火规则完善 → 7. 效果实现与局限性说明
