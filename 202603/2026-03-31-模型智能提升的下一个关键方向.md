# 模型智能提升的下一个关键方向

## TL;DR
AI 工程转向构建 Harness 系统，决定 Agent 性能上限。通过捕获执行轨迹实现“训练即部署”闭环，模型与系统共生演化。未来将向 Coordination Engineering 演进。

## Summary
**核心主题：**
随着基模能力趋于成熟，AI 工程的焦点已从模型本身转移到构建外部 "Harness"（驾驭系统）。Harness 不仅决定了 Agent 的性能上限，还通过捕获高质量的 "执行轨迹" 成为关键的数据资产，实现了 "训练即部署" 的闭环优化。

**关键信息与逻辑脉络：**

**1. 演进背景：从 Prompt 到 Harness**
*   **发展历程：** AI 工程经历了 Prompt Engineering（关注如何表达）、Context Engineering（关注信息供给）到 Harness Engineering（关注系统构建）三个阶段。
*   **驱动因素：** 随着模型（如 Claude Opus 4.5）智力瓶颈突破，限制 Agent 的因素转为系统稳定性、工具调用及反馈机制，即 "Harness"。
*   **核心定义：** Agent = LLM + Harness。Harness 包含运行环境、工具、记忆、评估与回滚机制等。

**2. 系统架构：Harness 的 6 个关键组件**
*   **信息层：** Memory & Context management（解决 Agent 看到什么）。
*   **执行层：** Tools & skills（扩展能力）、Orchestration & Coordination（任务拆解与调度）、Infra & Guardrails（运行环境与安全）。
*   **反馈层：** Evaluation & Verification（验证闭环）、Tracing & Observability（过程透明化）。

**3. 设计原则：分层优化策略**
*   **信息层（资源准备）：** 核心是 "精准优于求全"。
    *   采用渐进式披露（分层加载信息，避免 Context 衰减）。
    *   工具少而精（防止决策瘫痪）。
    *   控制 Context window 利用率在甜蜜区间（约 60% 以下）。
    *   利用 Subagent 进行 Context 隔离。
*   **执行层（任务规划）：**
    *   将研究、计划、执行、验证拆分为独立 Session。
    *   人类介入点应从事后审核前移至事前规划。
*   **反馈层（复利飞轮）：**
    *   构建反馈闭环，将失败经验转化为系统改进（如更新 AGENTS.md）。
    *   通过自动验证和迭代机制提升输出质量。

**4. 核心洞察：模型与 Harness 的共生关系**
*   **Harness 即数据：** Deepmind 工程师提出 "The Harness is the Dataset"，竞争优势在于捕获高质量的执行轨迹（决策过程、工具使用、反馈）。
*   **训练即部署：** 在 Agentic RL 中，Harness（训练环境）与模型共同演化。高质量的环境决定了模型能力的上限，模型会逐步内化 Harness 的能力。
*   **竞争格局：** 头部模型公司端到端优化 "模型 + Harness"，应用公司则基于开源模型和自有 Harness 进行垂直场景优化。

**5. 未来展望与创业机会**
*   **演进趋势：** 下一阶段将是 Coordination Engineering，即管理多 Agent 网络的协同工作，最终走向 Intention Engineering（人类仅设定目标）。
*   **创业方向：**
    *   信息层：企业知识库转动态上下文（如 Edra）。
    *   执行层：工作流编排、权限治理、沙箱基础设施（如 Temporal, Oasis, Daytona）。
    *   反馈层：Observability 与 Evaluation 平台（如 Braintrust）。
