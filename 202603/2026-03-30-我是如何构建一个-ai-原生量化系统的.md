# 我是如何构建一个 AI 原生量化系统的
- URL: https://www.bmpi.dev/dev/ai-native-investment-system/
- Added: 2026-03-30 02:18:03
- Tags: #trade #agent

## TL;DR
本文介绍了 AI 量化系统 "策引" 的构建经验。系统采用 DSL 和 DAG 架构，让 AI 充当翻译器而非决策者，通过混合 Agent 模式平衡模糊性与确定性，旨在建立透明可信任的决策框架。

## Summary
**核心主题**
本文分享了构建 AI 原生量化系统 "策引" 的实战经验，核心旨在解决投资者因不理解策略逻辑而在市场波动中产生情绪化决策的问题。系统致力于通过 AI 翻译与结构化表达，让用户建立对策略的信任与理解，而非单纯依赖 AI 给出买卖信号。

**关键信息与逻辑脉络**

1.  **设计理念：反向选择**
    *   区别于主流 "AI 替人做决策" 的路线，策引选择让 AI 充当 "翻译器"，将用户自然语言意图转化为可执行、可验证、可复现的结构化配置。
    *   核心目标是策略的透明与可解释性，唯有理解才能在回撤时保持纪律，建立信任。

2.  **技术选型：为何是 DSL**
    *   **拒绝代码生成**：Python 等通用代码存在幻觉、前视偏差、不可复现及高沙盒成本等问题，且普通用户难以理解。
    *   **拒绝 API 调用**：REST API 等接口在表达复杂计算图（如嵌套逻辑）时，参数结构会变得极其臃肿，表达力不足。
    *   **采用 DSL (Domain Specific Language)**：结合 API 的安全性与代码的表达力，由预定义的 "原语" 积木组合而成，AI 仅负责组合逻辑。

3.  **DSL 核心架构设计**
    *   **正交积木**：原语设计遵循单一职责原则（如 EMA, GreaterThan），通过自由组合覆盖复杂场景，而非设计大而全的组件。
    *   **DAG 执行模型**：策略逻辑表现为有向无环图（DAG），节点为原语，边为依赖关系，而非顺序执行的代码，利于 AI 生成与并行处理。
    *   **Schema 供应链**：Schema 是唯一事实源，通过 CI 自动生成 LLM 文档，确保 AI 读取的说明书与引擎执行的代码严格同源，避免漂移。

4.  **原语系统的演化**
    *   从基础技术指标与逻辑组合起步。
    *   引入 **Streak 原语**：解决需要跨 bar 记忆状态的需求（如连续确认），有效过滤假信号。
    *   引入外部数据源：支持 VIX、SPX 等市场指标，实现信号源与交易标的的解耦。
    *   承认 DSL 不是银弹：对于极其复杂的状态机逻辑，保留 Python 代码路径，避免陷入 DSL Hell。

5.  **Prompt 工程的演进**
    *   **早期困境**：仅提供原语清单导致失败率高，AI 常犯内联定义或参数错误。
    *   **绝对禁止项**：总结常见错误并以负面约束形式明确告诉 AI "不能怎么做"，效果优于正向引导。
    *   **Few-shot 验证**：引入 12 个从简到繁、经过回测验证的真实策略示例，供 AI 模仿结构。
    *   **闭环反馈**：将回测日志反馈给 AI，形成 "生成-验证-修正" 的闭环。

6.  **Agent 悖论与混合架构**
    *   **悖论**：如何平衡用户意图的模糊性与金融计算的确定性。
    *   **混合架构**：将 Agent 拆分为两层。
        *   **Local Agent (Orchestrator)**：运行在浏览器，负责处理模糊意图、维护 UI 状态和灵活编排。
        *   **Remote Agent (Processor)**：运行在服务端，负责确定性的重计算、回测流水线和数据分析。
    *   **演进路径**：从早期的单体混乱，到 Plan-Execute 模式，最终稳定为 Local 处理模糊性、Remote 处理工作量的 CLI Pattern。

7.  **核心原则与反思**
    *   **约束优于修复**：预先定义好 AI 可用的积木边界，将错误拦截在生成阶段。
    *   **同源性**：文档与执行逻辑必须源自同一份 Schema。
    *   **分层隔离**：确定性层（队列、回测、状态机）与概率性层（LLM 推理、文本生成）必须物理分离。
    *   **系统目标**：在高风险决策中，系统的价值在于帮助人类建立一套能理解、能信任、能执行的决策框架，而非直接提供答案。
