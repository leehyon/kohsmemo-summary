## TL;DR
AGENTS.md 为 AI Agent 提供非显性约束，避免"随机鹦鹉"困境。通过标准化结构，包含架构决策、技术债边界和部落知识，提升开发效能，减少无效搜索与 Token 消耗。

## Summary
核心主题：在企业级私有代码库中通过标准化 AGENTS.md 文件，为 AI Agent 提供精准的非显性约束，避免"随机鹦鹉"困境，提升开发效能。

关键信息与逻辑脉络：

1.  问题背景与战略意义
   - 企业私有库中 AI Agent 易陷入"随机鹦鹉"困境，不理解复杂架构逻辑
   - AGENTS.md 是成本控制与效能战略，非简单工程习惯
   - 研究数据支持：人为编写的上下文文件可提升任务成功率约 4%，而 LLM 自动生成会导致性能退化 3%
   - 盲目使用 LLM 生成上下文会引发"上下文腐败"，增加推理成本超过 20%
   - "部落知识"对 Agent 至关重要，但无法通过静态扫描获取

2.  核心编写原则：规避"显性冗余"
   - 高质量上下文优于海量上下文，只提供代码不可推断的知识
   - 区分显性描述与非显性约束：
     - 严禁罗列文件树或简单类说明，属于 Token 浪费
     - 不灌输通用编程准则，类比资深工程师任务分配
   - 必须包含的三类非显性信息：
     - 架构决策背景（"Why"）
     - 技术债边界
     - 确定性偏好

3.  标准化结构框架
   - 项目全景与技术栈：明确环境底座，防止错误 API 假设
   - 关键执行命令：置于文档头部，尽早纠正 Agent 惯性假设
   - 开发约定与风格：使用"错误 vs. 修正"代码块，示例胜过描述
   - 显性边界与禁区：设定物理与逻辑红线，确保私有库安全
   - 部落知识快照：捕捉无法从文档或代码中推导出的事实

4.  层次化上下文分发策略
   - 超大型代码库采用渐进式披露原则，避免注意力层稀释
   - 根目录 AGENTS.md：定义全局底座、环境约束、红线禁区，最高缓存优先级
   - 子目录 AGENTS.md：定义具体业务逻辑、领域特定约束，中等缓存优先级
   - 多层级配置可显著减少 Agent 无效搜索，降低平均 Token 消耗 20% 以上

5.  编写技巧与质量控制
   - "200 行原则"：文档必须精炼，超过 200 行通常意味着文件已失效
   - 基于"失败驱动"的动态更新流程：任务失败→根因分析→增补约束→本地回滚→回归验证
   - 符号链接与跨平台策略：将 CONTRIBUTING.md 软链接至 AGENTS.md，确保单一事实来源

AGENTS.md 示例包含项目概述、关键命令、编码约定、边界规则和部落知识五个核心部分，体现了精炼、实用的特点。
