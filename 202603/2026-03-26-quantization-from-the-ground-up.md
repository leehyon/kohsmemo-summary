# Quantization from the ground up
- URL: https://ngrok.com/blog/quantization
- Added: 2026-03-26 02:00:58
- Tags: #llm #math

## TL;DR
本文解析 LLM 量化原理，通过降低数值精度实现模型缩容与提速。介绍了非对称量化及分块处理 Outliers 策略。实测表明，4-bit 量化性能接近原模型，而 2-bit 效果极差。

## Summary
**核心主题**：解析 LLM 量化原理，阐述如何通过降低数值精度将模型体积缩小 4 倍、推理速度提升 2 倍，同时将精度损失控制在可接受范围内。

**模型规模与数字存储**
*   **模型巨大的成因**：LLM 由数十亿个 Parameters (参数) 或 Weights (权重) 构成，这些参数构成了具有多层和密集连接的巨大运算图。
*   **浮点数存储机制**：计算机使用 Floating Point (浮点数) 表示小数，通过符号位、指数位和尾数位权衡表示范围与精度。
*   **参数分布特征**：Model parameters (模型参数) 通常数值较小且集中在 0 附近，这与浮点数在高精度区域的密集分布相吻合，为压缩提供了空间。

**量化方法演进**
*   **更小的浮点格式**：使用 Float16、bfloat16 (Google Brain)、Float8 甚至 Float4 替代标准的 Float32，以牺牲部分精度换取更小的内存占用。
*   **简单舍入的缺陷**：直接将 Float16 舍入到 Float4 会导致许多权重归零，切断输入到输出的路径，从而破坏模型功能。
*   **对称量化**：通过缩放因子将参数映射到围绕 0 的对称整数区间。该方法实现简单，但当参数分布不对称（如正负范围差异大）时会浪费存储空间，导致较高误差。
*   **非对称量化**：引入 Zero Point (零点) 偏移量，根据数据的实际最大值和最小值动态调整映射区间。相比对称量化，它能更高效地利用有限的位数，显著降低平均误差。

**实际应用与质量评估**
*   **分块量化**：由于模型中存在少量 Outliers (异常值)，全局量化会破坏整体效果。实际应用中将参数分块（如每块 32-256 个），独立计算缩放因子和零点，将异常值的影响限制在局部。
*   **困惑度**：衡量模型对正确 Token 的预测置信度。测试显示，4-bit 量化导致的困惑度上升较小，而 2-bit 量化会导致模型置信度崩塌。
*   **KL 散度**：比较原始模型与量化模型输出概率分布的重合度。相比困惑度，它能更全面地捕捉量化对模型行为的影响。
*   **基准测试与实测**：在 GPQA 等基准测试中，8-bit 和 4-bit 模型表现接近原始模型，甚至有时因噪声表现更好；但 2-bit 模型几乎完全失效，陷入死循环或无法回答。实际对话测试也证实 4-bit 模型能正常回答事实性问题，而 2-bit 模型只会输出乱码。
