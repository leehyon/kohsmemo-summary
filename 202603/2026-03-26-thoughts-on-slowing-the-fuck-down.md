# Thoughts on slowing the fuck down
- URL: https://mariozechner.at/posts/2026-03-25-thoughts-on-slowing-the-fuck-down/
- Added: 2026-03-26 01:57:50
- Tags: #agent

## TL;DR
盲目依赖 AI Agent 会导致软件质量崩塌与复杂性失控。开发者应回归人类主导，主动降速，仅将 Agent 用于辅助。通过手写核心代码保持掌控，最终构建高质量、可维护的软件系统。

## Summary
*   **核心主题**
    软件开发中 AI Agent 的局限性以及回归人类主导、慢节奏开发模式的必要性。

*   **关键信息**
    盲目依赖 AI Agent 生成代码会导致软件质量崩塌与系统复杂性失控；开发者应通过“慢下来”来保持对代码库的理解与掌控，利用 Agent 而非被其替代。

*   **逻辑脉络**
    *   **现状观察**
        引入 AI Agent 后，软件行业普遍出现质量下降，故障频发，开发者陷入了缺乏代码审查与设计思考的“加速陷阱”。
    *   **问题剖析**
        *   **错误累积**：Agent 无学习能力，且缺乏人类作为瓶颈的自然限制，导致微小错误在短时间内以指数级复合成灾难性后果。
        *   **复杂性贩卖**：Agent 倾向于引入过度复杂的架构与冗余抽象，在几周内即可重现人类企业级代码库需数年才能形成的“烂摊子”。
        *   **搜索召回率低**：代码库的混乱与膨胀导致 Agent 无法精准定位所需代码，进而引发更多重复与不一致的修复尝试。
    *   **解决方案**
        *   **合理分工**：将 Agent 用于范围明确、有明确评估函数、非关键性或探索性的任务。
        *   **人类把关**：人类必须作为最终的质量裁决者，对 Agent 产出进行评估与定稿。
        *   **主动降速**：限制 Agent 生成代码的数量以匹配审查能力，坚持手写架构与核心代码以保留对系统的感知力与摩擦感带来的学习机会。
    *   **最终目标**
        构建可维护、高质量的软件系统，保持人类对技术的掌控权与自主权。
