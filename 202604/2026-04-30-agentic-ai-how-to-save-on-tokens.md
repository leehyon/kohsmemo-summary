# Agentic AI: How to Save on Tokens

## TL;DR
本文探讨生产环境 Agentic AI 的成本优化，提出复用 Token 、按需加载、模型路由及上下文清理等策略，旨在降低 Token 消耗并平衡性能与成本。

## Summary
**核心主题**

本文探讨了在生产环境中构建 Agentic AI 时，如何通过设计优化显著降低 Token 消耗与运营成本，并分析了不同优化策略的适用场景与权衡。

**逻辑脉络**

1.  **问题陈述**：指出 Agent 随着上下文（如系统提示词、工具定义、对话历史）的增长，Token 成本呈指数级上升的现象。
2.  **设计原则**：提出四大核心优化原则，旨在通过复用、精简、路由和清理来控制成本。
3.  **技术深挖**：详细解析每种原则背后的机制（如 K/V 缓存、语义匹配、级联验证等）及实际应用案例。
4.  **综合建议**：根据具体用例（如长系统提示词、通用问答、复杂任务）推荐组合策略。

**关键信息**

*   **尽可能复用 Token**
    *   **Prompt Caching (提示缓存)**：利用 K/V (Key/Value) 缓存机制跳过预填充 步骤。要求提示词前缀必须完全匹配。API 提供商（如 OpenAI, Anthropic）对缓存部分的输入 Token 提供高达 90% 的折扣。
    *   **Semantic Caching (语义缓存)**：基于 Embeddings 匹配含义相似的请求，直接返回历史结果。适用于存在大量重复通用问题的 Q&A 场景，但需解决时效性 (TTL)、用户隔离及错误缓存等工程挑战。

*   **不预加载休眠 Token**
    *   **保持上下文精简**：将基础系统提示词控制在最小且稳定的范围，仅在需要时按需加载详细信息或工具描述，以避免干扰 Prompt Caching 的命中率。
    *   **工具搜索**：使用搜索工具代替在上下文中直接列出数百个工具定义，减少初始 Token 消耗并提高模型选择工具的准确性。

*   **廉价任务使用廉价模型**
    *   **Routing (路由)**：在生成前评估任务难度或意图，将简单请求分发至小型或低成本模型。挑战在于路由模型的准确性可能影响整体质量。
    *   **Cascading (级联)**：先让廉价模型尝试生成，若置信度（如 logprobs）低则升级至大型模型。适用于大多数问题较简单的场景。
    *   **Subagents (子代理)**：将特定子任务（如代码库搜索）委托给使用较小模型的独立代理，既能节省成本又能保持主上下文清洁。

*   **保持上下文清洁**
    *   **清理垃圾数据**：主动移除无用的工具输出、冗长日志、重复文件读取和死胡同尝试，仅保留对当前任务有意义的状态。
    *   **上下文压缩**：通过摘要和归档机制压缩对话历史。研究表明，良好的压缩策略可减少 30-70% 的 Token 预算，并因减少噪声干扰而提升任务解决率。
