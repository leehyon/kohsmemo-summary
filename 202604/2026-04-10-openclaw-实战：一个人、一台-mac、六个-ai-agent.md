## TL;DR
本文分享基于 OpenClaw 构建生产级 AI 系统的实战经验。通过解决上下文管理、记忆进化及多 Agent 协作三大难题，实现了系统的 7x24 稳定运行与自主迭代。

## Summary
**核心主题**
本文分享了基于 OpenClaw 框架构建一套由 1 个编排者、5 个专业 Agent 和 6 个 ACP 编码专家组成的生产级 AI 系统的工程实战经验。重点阐述了如何通过系统化的工程设计，解决 Agent 从“能聊天”进阶到“能干活”过程中的上下文管理、记忆进化与多 Agent 协作三大核心问题，实现系统的 7x24 稳定运行与自主迭代。

**系统架构与团队阵型：1+5+6**

*   **Zoe (大龙虾/CTO)**：系统总编排者。负责技术方案设计、任务分发、每日巡检、记忆压缩及 Tech Radar 评估。自主设计了三态通信协议和 Task Watcher。
*   **AI 哨兵**：情报中枢。采集 GitHub Trending、arXiv 等 100+ 信息源，产出晨报与趋势分析，主动评估新技术对系统的影响并推动落地。
*   **交易蜘蛛**：量化分析师。利用 20 个原子工具和四步分析框架进行 A 股与美股分析，生成评分与止损建议。
*   **Macro (首席经济学家)**：提供宏观因子映射。数据驱动分析宏观对市场的影响，为 Trading 提供决策输入。
*   **内容蜘蛛**：内容策略师。基于情报链进行创意生成与写作，自主调研工具去“AI 味”，设计了五篮子热点雷达。
*   **管家蜘蛛**：生活助理。深度集成 Apple 生态，负责日程、健康提醒及生活服务。
*   **ACP 编码专家**：包括 Claude Code、GPT-5.3-Codex 等 6 种模型，通过 ACP 协议按需并发执行编码任务。

**三大核心工程挑战与解决方案**

1.  **上下文是 Agent 的操作系统**
    *   **问题**：Session 无限膨胀导致系统崩溃 (P0)，或因信息过载导致关键规则失效 (P2)。
    *   **解决方案**：双层控制机制。
        *   **Context Engineering**：设计信息架构，SOUL.md 定义核心约束，AGENTS.md 定义协议，Skills 按需加载，区分冷热数据。
        *   **Harness 自动管理**：配置 Compaction (提取精华)、Context Pruning (裁剪旧上下文)、Session Reset (定时重置) 及 Maintenance (自动清理)，形成防御闭环。

2.  **让 Agent 真正“记住”并“成长”**
    *   **问题**：Agent 反复犯同样错误，无法从经验中进化。
    *   **解决方案**：五层记忆模型 + 自主迭代循环。
        *   **分层存储**：L1 身份层 (SOUL.md) -> L2 长期记忆 (MEMORY.md) -> L3 中期记忆 -> L4 短期记忆 (.learnings/) -> L5 持久化。
        *   **迭代流程**：触发事件 -> .learnings/ 即时记录 -> 每日反思 Cron 评估 -> 频繁经验 Promote 到 MEMORY.md -> 下次 Session Bootstrap 注入 -> 行为改进。
        *   **案例**：Content 自主调研去 AI 味工具并发布为 Skill；Zoe 发现 ReMe 框架并自主评估落地。

3.  **多 Agent 协作是协议问题，不是群聊问题**
    *   **问题**：Agent 在群聊中陷入“收到/确认”的死循环 (ACK Storm)，协作效率低且状态不可控。
    *   **解决方案**：协议级设计与状态共享。
        *   **三态协议**：定义 [request] -> [confirmed] -> [final] 状态机，Final 后强制静默，终结无效对话。
        *   **Shared-Context**：使用共享文件系统（如 tech-radar.json, intel/）作为异步状态传递介质，替代不可靠的消息传递。
        *   **Task Watcher**：Zoe 自主设计的异步监控系统，解决“Agent 说会做但实际没做”的问题，通过 Cron 轮询任务状态并触发告警。

**实战成果与关键指标**
*   **运行规模**：52 个 cron 任务，118 个 Skills，每天数千次 LLM 调用，2086 行运维脚本。
*   **自主进化**：Agent 自主设计并沉淀了通信协议、Task Watcher、去 AI 味工具等核心能力，实现了从“执行者”到“设计者”的跃迁。
*   **稳定性**：通过记忆维护、心跳检测、自愈脚本，实现了半个月内 23 次自动恢复，系统在无人干预下持续产出。

**关键经验总结**
*   **工程主导**：90% 的工作量在解决上下文膨胀、通信协议等工程问题，而非单纯优化 Prompt。
*   **可预测性优先**：在生产环境中，“智能”往往导致不可控（如智能压缩丢失数据），显式规则和硬约束优于软性建议。
*   **反退化机制**：系统必然退化（熵增），必须建立 Compaction、Maintenance、巡检等多层反退化机制。
*   **协议重于模型**：协作效率取决于协议设计（如三态机），而非模型智商。
*   **循序渐进**：建议从 1 个 Agent 开始，先稳运行再扩规模，重视 SOUL.md 的精简和 Session 参数的初始设置。
