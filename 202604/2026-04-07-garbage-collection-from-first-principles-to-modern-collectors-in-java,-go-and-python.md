## TL;DR
本文解析 GC 理论演进，对比引用计数与追踪法，深入剖析 Go 语言基于三色标记与混合写屏障的并发实现，旨在最小化 STW 停顿。

## Summary
*   **核心主题**
    本文深入探讨了垃圾回收 的演进历程，从 McCarthy 和 Wilson 的理论基础出发，解析了引用计数与追踪 的核心算法原理，并重点剖析了现代语言 (特别是 Go) 中并发垃圾回收的实现机制与工程权衡。

*   **关键信息与逻辑脉络**

    1.  **历史与理论基础**
        *   **McCarthy (1960)：** 在 Lisp 中首次提出标记-清除 算法，实现了自动内存管理，但要求 Stop-The-World (STW)，即程序必须完全暂停以进行回收。
        *   **Wilson (1992)：** 系统化了 GC 理论，提出了两大关键观察：**分代假说** (Generational Hypothesis，即大多数对象朝生夕灭) 和 **三色标记** (Tricolor Marking)，后者为实现并发回收提供了理论抽象。

    2.  **两种基本方法**
        *   **引用计数：** 每个对象维护引用次数，归零即释放。
            *   *优点：* 回收具有确定性，无需 STW。
            *   *缺点：* 无法处理循环引用；每次指针赋值均有性能开销 (多线程下需原子操作)。
        *   **追踪 (Mark-and-Sweep)：** 从根集合 遍历对象图，标记存活对象。
            *   *优点：* 自然处理循环引用。
            *   *缺点：* 存在内存碎片；基础的标记-清除需要长时间 STW。

    3.  **追踪算法的变体**
        *   **标记-清除：** 简单，但会导致内存碎片化。
        *   **复制：** 将堆分为两半，只复制存活对象。分配极快且无碎片，但空间利用率仅为 50%。
        *   **标记-整理：** 移动存活对象以消除碎片，无空间浪费，但移动成本较高。
        *   **分代回收：** 基于分代假说，将堆分为年轻代 和老年代。年轻代使用复制算法频繁回收，老年代使用标记-整理低频回收。

    4.  **现代 GC 实践：Go 的并发实现**
        *   **架构：** 非分代、非整理、并发标记。目标是最小化 STW 停顿时间。
        *   **三色标记机制：** 将对象分为白色 (未访问/垃圾)、灰色 (已访问但子对象未扫描)、黑色 (已处理)。
        *   **写屏障：** 为解决并发标记中用户代码修改指针导致的 "黑对象指向白对象" 问题，引入了写屏障。
            *   **混合屏障：** 结合了 Dijkstra 插入屏障 和 Yuasa 删除屏障。在堆指针写入时同时对新旧引用进行着色，从而避免了在标记结束时对栈 的重新扫描，显著降低了 STW 时间 (从毫秒级降至亚毫秒级)。
        *   **调优与监控：**
            *   `GOGC`：控制堆增长率，默认值为 100 (即堆增长一倍后触发 GC)。
            *   `GOMEMLIMIT`：设置软内存上限，适用于容器环境。
            *   `GODEBUG=gctrace=1`：用于输出详细的 GC 运行日志，分析 STW 时间与 CPU 开销 (STW、并发辅助、后台 GC)。
