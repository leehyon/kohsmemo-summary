## Summary
一、核心主题
基于个人投入超万元拥抱 AI 一年的深度实践，分享了从工具链搭建、工程范式演进到知识管理模式的务实经验，并探讨了对 AI Native 形态、人机关系及技术发展趋势的思考。

二、关键信息与逻辑脉络

1. 务实落地：工具、实践与方法论
   - **MAC 工具链升级**：为应对多任务并行及海量 AI 工具的使用，重构桌面环境。核心组件包括 Raycast（启动器）、AeroSpace（分屏）、Claude Code（核心 Agent）及自研 Cockpit（跨机器 Agent 任务状态监控），以提升对并发任务的管理能力。
   - **摘取低垂果实**：利用 Agent 能力轻松完成以往繁琐之事。
     - 快捷指令自动化生成与维护。
     - 自动诊断并提交 GitHub PR 优化开源项目。
     - 全自动托管参加 Kaggle 比赛，曾获榜单第六名。
   - **工程范式演进**：从单一 Prompt 向系统性 Harness 转变。
     - Prompt Engineering：基础指令撰写，难以应对复杂长时任务。
     - Context Engineering：优化 Agent 推理时的信息环境，精准控制上下文。
     - Spec-driven Development (SDD)：先写契约定义需求与验收标准，大幅减少返工。
     - Harness Engineering：构建约束体系防止熵增。包含约束层（机械化规则）、文档层（精简索引）、反馈层（错误转能力）及清理层（自动对抗技术债）。
   - **知识模式重构**：让 Agent 替代人类进行高效学习与集成。
     - 痛点：技术迭代过快，“古法学习”（人工收集）无法应对信息爆炸。
     - 方案：基于 Claude Code 的 Skills 体系构建知识循环。
     - 实践：利用 ai-news 聚合、podcast-batch 转录、ai-practices 提炼等技能，由 Agent 扫描资讯并沉淀最佳实践，反哺工作流。

2. 务虚思考：碎碎念与观点
   - **AI Native 的形态**：手动撰写文章并不符合 AI Native 特性，当前围绕 Claude Code 构建工具链可能仅是过渡，未来理想的交互形态可能更接近深度定制的即时通讯或语音应用。
   - **与 AI 的关系**：接受“Jagged Intelligence”（能力参差不齐）的现状，通过高频使用建立直觉。建议将 AI 视为“好朋友”，以“最强模型”加“持续交流”作为最高效的学习路径。
   - **阶段性反思**：回顾早期使用 AI 的经历，曾面临网络限制、支付门槛及效率反而降低的困境。但随着大模型能力的飞跃，现阶段钻研 AI 已能带来确定的正收益。
