# 让 Claude Code 在你睡觉时持续运行：完整实战指南

## TL;DR
本指南详述如何配置 Claude Code 实现无人值守自动化编程。通过设定运行模式、Ralph Wiggum 循环、安全 Hook 及 Docker 隔离环境，使 AI 能够在夜间自动执行代码编写与修复。

## Summary
核心主题
本文是一份关于配置 Claude Code 实现长时间、无人值守自动化编程的实战指南，旨在通过特定的技术设置，让 AI 在夜间自动执行代码编写、测试及修复任务。

关键信息

1.  **三种自主运行模式**
    *   **`-p` 标志**：非交互式运行，接收指令执行完成后退出，适合低内存服务器，是自动化的核心。
    *   **`--permission-mode auto`**：自动批准安全操作并拦截高风险操作，兼顾安全与效率。
    *   **`--dangerously-skip-permissions`**：完全跳过权限确认，效率最高但风险极大，必须在容器（Docker）中运行。

2.  **核心执行策略：Ralph Wiggum 循环**
    *   利用 Bash `while` 循环持续向 Claude 喂入详细的 `PROMPT.md` 文件。
    *   AI 每次循环查看文件状态和 Git 历史，选择任务、执行并提交。
    *   **关键点**：Prompt 文件需包含架构说明、目标和明确的完成标准，而非简单的一句话。

3.  **四大防护 Hook（防止过夜灾难）**
    *   **No-Ask-Human**：阻止 AI 提问，强制其自主决策，防止会话挂起。
    *   **Context Monitor**：监控上下文使用量，在接近阈值时自动触发压缩。
    *   **Syntax Check**：编辑文件后立即运行语法检查，防止错误级联。
    *   **Decision Warn**：拦截 `rm -rf`、`git push --force` 等破坏性命令及对受保护分支的操作。

4.  **环境持久化与基础设施**
    *   **Tmux**：Claude Code 需要 TTY，必须使用 tmux 保持会话，支持分离与重连。
    *   **运行环境**：推荐 VPS + Tailscale 方案确保 7×24 运行；本地 macOS 需使用 `caffeinate` 防止休眠。
    *   **安全容器**：必须在 Docker 容器中运行（建议配合 `--network none`），防止 AI 删除宿主机文件。

5.  **上下文与成本管理**
    *   **上下文耗尽应对**：采用“检查点/交接模式”，在上下文压缩前将当前状态（已完成、下一步、受阻）写入外部文件（如 `mission.md`）。
    *   **配置精简**：`CLAUDE.md` 应控制在 200 行以内以节省 Token。
    *   **成本控制**：设置 `--max-budget-usd` 硬上限；常规任务使用 Sonnet 模型以降低成本；利用夜间非高峰时段避开速率限制。

6.  **CI/CD 集成与 Cron 任务**
    *   支持通过 GitHub Action 进行代码审查。
    *   Cron 任务需通过 `state.md` 文件维持状态，使用 `--bare` 模式（无 hooks/MCP）确保执行的可复现性和速度。

逻辑脉络
1.  **模式选择**：根据安全需求选择运行模式（推荐 `-p` 结合工具白名单），并设置最大迭代次数与预算上限。
2.  **策略构建**：编写包含详细上下文和决策规则的 `PROMPT.md`，构建 Ralph Wiggum 循环实现持续工作流。
3.  **安全加固**：安装四个关键 Hook，拦截危险操作并防止会话卡死，确保在无人工干预下能处理异常。
4.  **环境部署**：利用 tmux 和 Docker 搭建持久化隔离环境，配置防止休眠策略，确保进程持续运行。
5.  **资源优化**：实施检查点机制应对上下文窗口限制，通过模型切换和非高峰时段调度优化 Token 成本与速率限制。
6.  **自动化集成**：将流程集成到 Cron 或 CI/CD 中，遵循检查清单启动无人值守任务，次日通过 Git Log 检查产出。
